Specifier

A person who is creating the implementation guide or message profile.

Slicing

Defines a set of different constraints that are possible for a field instance (occurrence).

Base Data Type

Slicing

this is the HL7 base datatype for the field as defined in Chapter 2 of the base standard. The default datatype and all the datatypes flavors associated with that field must derive from the base datatype.

Default Data Type

Slicing

this is the data type flavor that is triggered when the parameter to identify a slice are not satisfied. The default datatype is optional. If the default is not specified, then a field occurrence instance must contain a discriminator value that is explicitly defined.

Discriminator Position

Slicing

this is the position of the discriminator element(s). The path is relative to the base datatype. All occurrences of the field have the same discriminator position. Discriminator position can be one or more elements (e.g., PID-11.6 (Country) and PID-11.7 (Address Type)).

Discriminator Value

Slicing

this is the value of the discriminator. For fixed value, the evaluation is on a literal value. For exists, the evaluation is on Boolean logic. For pattern, the evaluation is on a regex (regular expression).

Slice Min

Slicing

this attribute determines the minimum number of occurrences that must contain "discriminator value" and therefore conform to “slice datatype flavor”.

Slice Max

Slicing

this attribute determines the maximum number of occurrences that must contain "discriminator value" and therefore conform to “slice datatype flavor”.
